현재 작업 중인 태스크를 완료하고 PR을 생성합니다.

## 1. 변경사항 분석

`git diff --stat` 과 `git diff` 로 변경사항을 확인하고, 어떤 파일이 어떻게 바뀌었는지 파악합니다.

## 2. 린트 & 테스트

- 백엔드: `uv run ruff check .` 린트 확인
- 프론트엔드: `pnpm lint` 린트 확인
- 테스트가 있으면 실행

## 3. 커밋 메시지 자동 생성

변경사항을 분석하여 Conventional Commits 형식의 커밋 메시지를 자동 생성합니다.

규칙:
- 제목: `{type}({scope}): {한글 요약}` (50자 이내)
  - type: feat, fix, refactor, chore, docs, test, style
  - scope: 변경 대상 모듈/기능 (예: auth, chat, api)
- 본문: 변경 내용을 항목별로 정리 (한글)
- 이슈 연결: `Refs #이슈번호`

예시:
```
feat(oauth): Google OAuth2 로그인 구현

- /api/v1/auth/google 엔드포인트 추가
- GoogleOAuthService 서비스 레이어 구현
- JWT 토큰 발급 로직 연동
- 관련 단위 테스트 추가

Refs #42
```

**생성한 커밋 메시지를 사용자에게 보여주고, 수정할 부분이 있는지 확인합니다.**
사용자가 수정을 원하면 반영 후 커밋, "확인" 또는 "ㅇㅇ" 등 승인하면 그대로 커밋합니다.

## 4. 커밋 & Push

사용자 확인 후:
- `git add -A` 로 스테이징
- 확인된 메시지로 커밋
- `git push -u origin {현재브랜치}` 로 push

## 5. PR 메시지 자동 생성

커밋 히스토리와 변경 파일을 기반으로 PR 메시지를 자동 생성합니다.

PR 형식:
```
제목: {커밋 타입에 맞는 한줄 요약} (50자 이내)

## 개요
{이 PR이 무엇을 왜 하는지 2-3문장}

## 변경 내용
- {주요 변경사항 1}
- {주요 변경사항 2}
- {주요 변경사항 3}

## 테스트
- [ ] {테스트 항목 1}
- [ ] {테스트 항목 2}

Closes #{이슈번호}
```

**생성한 PR 메시지를 사용자에게 보여주고, 수정할 부분이 있는지 확인합니다.**
사용자가 수정을 원하면 반영 후 PR 생성, 승인하면 그대로 생성합니다.

## 6. PR 생성 & 이슈 업데이트

사용자 확인 후:
- `gh pr create --title "{제목}" --body "{본문}"` 으로 PR 생성
- 이슈에 "✅ PR 생성 완료" 코멘트 추가: `gh issue comment {이슈번호} --body "✅ PR #{PR번호} 생성 완료"`

선택적으로 $ARGUMENTS 에 추가 지시사항을 전달할 수 있습니다.
